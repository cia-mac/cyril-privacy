How Cyril Works

Temporal accumulation — blending time-lapse frames into layered motion trails

30

Cyril reads a sequence of time-lapse images and blends them with an exponential moving average. Each frame mixes into a running composite — recent frames dominate, older frames decay. The result: long-exposure trails where motion accumulates over time.

EMA update (per frame):
composite = composite × (1 - w) + new_frame × w

Weight from trail length:
w = 1 - eln(0.10) / trail

After trail frames, contribution decays to 10%.

3. Accumulation in Action

A colored dot traces a path. Each frame, the canvas blends via EMA — exactly how Cyril processes pixels. Increase the trail to see longer persistence; decrease it for sharper, shorter trails.

Each pixel: composite = composite × (1-w) + frame × w. Where the dot was, it decays. Where it is, it brightens. The trail length controls how fast the past fades.

4. Short vs Long Trail

Same motion, different persistence. Short trails track the dot closely. Long trails smear its history across the canvas.

5. The Processing Pipeline

Every frame passes through this chain. All math runs in Float32 linear sRGB — no gamma artifacts. The sRGB conversion happens only at output via a 4096-entry LUT.

Color corrections run on GPU via CIFilter. Accumulation and corrections use vDSP/Accelerate for SIMD-optimized batch ops. No 8-bit bottleneck anywhere in the chain.
