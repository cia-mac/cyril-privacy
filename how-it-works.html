How Cyril Works

Temporal accumulation — turning time-lapse sequences into ghostly, layered motion

The Core Idea

Cyril reads a sequence of time-lapse images (hundreds or thousands of frames from a locked camera) and blends them using an exponential moving average. Each new frame is mixed into a running composite — recent frames contribute strongly, older frames fade away gradually. The result: ghostly trails of motion, like a long exposure that remembers.

For each new frame:
composite = composite × (1 - w) + new_frame × w

Weight from trail length:
w = 1 - eln(0.10) / trail

Meaning: after trail frames, a pixel decays to 10% of its original contribution.

4. Short vs Long Trail

Short trails = sharp, recent motion. Long trails = deep, accumulated history.

Trail = 5 (responsive)

Only the last ~5 frames matter. Motion is crisp, trails are short.

Trail = 80 (deep memory)

Dozens of frames contribute. Motion smears into long, ghostly trails.

5. The Pipeline

For each frame in the sequence:
1. Load image → decode to Float32 linear sRGB pixels
2. Apply color correction (8 CIFilter parameters, GPU)
3. Apply exposure normalization (EXIF-based EV matching)
4. Apply deflicker (Gaussian-smoothed luminance)
5. EMA blend into running composite
6. Convert via sRGB LUT → write to video/image
The entire pixel pipeline runs in Float32 linear light — no gamma curve artifacts during blending. The sRGB conversion happens only at the final output stage via a 4096-entry lookup table. All vector math uses Apple's vDSP/Accelerate framework for SIMD-optimized batch operations.
