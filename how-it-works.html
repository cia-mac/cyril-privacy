How Cyril Works

Temporal accumulation — blending time-lapse sequences into layered motion trails

The Core Idea

Cyril reads a sequence of time-lapse images and blends them using an exponential moving average. Each new frame mixes into a running composite. Recent frames contribute strongly; older frames decay. The result: long-exposure-like trails where motion accumulates over time.

EMA update (per frame):
composite = composite × (1 - w) + new_frame × w

Weight from trail length:
w = 1 - eln(0.10) / trail

After trail frames, a pixel's contribution decays to 10%.

5. The Processing Pipeline

Every frame passes through this chain. All pixel math runs in Float32 linear sRGB — no gamma artifacts during blending. The sRGB conversion happens only at the final output via a 4096-entry lookup table.

All vector operations use Apple's vDSP/Accelerate framework for SIMD-optimized batch processing. Color corrections run on GPU via CIFilter. The entire pipeline avoids 8-bit bottlenecks.

7. Deflicker

Time-lapses often have per-frame brightness jitter from aperture/shutter inconsistency. Cyril measures each frame's luminance (Rec.709, trimmed mean of 10th-90th percentile), then Gaussian-smooths the luminance curve and applies corrections. Corrections are soft-clamped to ±2 stops via log-space tanh compression.
